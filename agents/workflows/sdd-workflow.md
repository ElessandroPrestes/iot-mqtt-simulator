# SDD Workflow para IoT MQTT Simulator

1. **Idea:** Nova funcionalidade (ex: novo tipo de sensor).
2. **Discovery:** Mapeamento do impacto na API e broker.
3. **Architecture:** Decisão (ex: novo tópico MQTT, schema Mongo).
4. **Specification:** Documento em `specs/`.
5. **Human Approval:** Aprovação por desenvolvedor.
6. **Tasks:** Quebra das atividades (API, Simulator, Vue).
7. **Implementation:** Código escrito nos serviços.
8. **Testing:** Testes (Jest/Vitest).
9. **Review:** Análise do código vs Padrões.
10. **Refactor:** Correções se necessário.
11. **Documentation:** Atualização do `PROJECT.md` e APIs.
12. **Release:** Fechamento.
