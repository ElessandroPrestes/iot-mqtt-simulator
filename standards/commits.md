# Padrões de Commits

O projeto segue as convenções do Conventional Commits.

Prefixos permitidos:
- `feat:` Nova feature
- `fix:` Correção de bug
- `chore:` Alterações que não impactam fonte de produto (ex: build, config)
- `docs:` Alteração de documentação (`PROJECT.md`, specs, knowledge base)
- `test:` Inclusão/correção de testes
- `refactor:` Refatorações sem alteração comportamental
